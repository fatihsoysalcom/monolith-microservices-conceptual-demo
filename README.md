# Monolith Microservices Conceptual Demo

This example demonstrates the core conceptual difference between a monolithic application and a microservices architecture. It shows how a single, tightly-coupled application handling multiple business domains can be conceptually broken down into smaller, independent services, each responsible for a specific domain. Note that this is a conceptual simulation within a single file, not a distributed system.

## Language

`python`

## How to Run

Save the code as `main.py`.
Run from your terminal using: `python main.py`
